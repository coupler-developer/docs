# Development Policies (개발 정책)

이 디렉토리는 Coupler 프로젝트의 개발 정책 문서를 저장합니다.

## 정책 문서 종류

### 코딩 표준
- 코드 스타일 가이드
- 네이밍 규칙
- 주석 작성 규칙

### 개발 프로세스
- Git 브랜치 전략
- 코드 리뷰 정책
- 배포 프로세스

### 보안 정책
- 인증/인가 처리 가이드
- 데이터 암호화 정책
- API 보안 가이드

### 아키텍처 가이드
- 시스템 아키텍처
- 데이터베이스 설계 원칙
- API 설계 가이드

## 정책 문서 작성 가이드

각 정책 문서는 다음 형식을 따릅니다:

1. **제목**: 정책의 명확한 제목
2. **목적**: 정책의 목적과 필요성
3. **적용 범위**: 어떤 프로젝트/팀에 적용되는지
4. **세부 내용**: 정책의 구체적인 내용
5. **예시**: 올바른/잘못된 예시
6. **참고 자료**: 관련 문서나 링크
