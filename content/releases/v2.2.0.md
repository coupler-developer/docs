## 릴리스 결과

- `coupler-api` 운영 EC2를 `4b96c904fe550f9295f0f4b3fb164814d6e468d3` 기준으로 배포하고 내부/외부 응답과 DB 연결을 확인했다.
- `coupler-mobile-app` Android/iOS Store 빌드는 `2.2.0 (97)` 승인 기준으로 `v2.2.0` 태그를 생성했다.
- `coupler-mobile-app` Android/iOS NextPush Production 배포를 `2.2.0` binary 대상으로 완료했다.
- Android/iOS 제출 기준 커밋이 달라 제출 마커는 platform별로 분리했고, 릴리스 기록으로 tag commit과 artifact/hash 증빙을 이관한 뒤 로컬/원격 태그를 삭제했다.
- `docs` GitHub Release와 `docs-site-v2.2.0.tar.gz` artifact를 생성했고, NextPush 배포 결과를 docs-only corrective reissue로 반영했다.
- `coupler-admin-web`, 운영 `RDS`는 이번 릴리스 범위에 포함하지 않았다.

## 메인 흐름

1. Mobile Store Android/iOS `2.2.0 (97)` 승인 상태를 확인했다.
2. 운영 `coupler-api`를 `origin/main`의 `4b96c904fe550f9295f0f4b3fb164814d6e468d3`까지 fast-forward 반영했다.
3. 운영 `coupler-api`에서 `pnpm install --frozen-lockfile`, `sudo pm2 restart coupler-api --update-env`, `sudo pm2 save`를 수행했다.
4. API 내부/외부 응답, DB `SELECT 1`, PM2 상태, 로그를 확인한 뒤 `coupler-api` `v2.2.0` 태그를 push했다.
5. `coupler-mobile-app` `v2.2.0` 태그를 승인된 Store 기준 커밋 `e4dd5b7c5d56b26065c6de31eca7b9c5f6225248`에 생성했다.
6. 제출 마커 태그의 artifact/hash 증빙을 이 문서에 이관하고, `coupler-mobile-app` 로컬/원격 제출 마커 태그를 삭제했다.
7. `coupler-mobile-app` Android/iOS NextPush Production 배포를 `2.2.0` binary 대상으로 수행했다.
8. `docs` `v2.2.0` 태그를 push해 GitHub Release와 site artifact를 생성했다.
9. NextPush 배포 결과를 반영하기 위해 `docs-only corrective reissue`로 `docs` Release 본문과 site artifact를 정정했다.
