## 릴리스 결과

- `coupler-api`는 개발계와 운영 EC2에 `0d3532141f82c8582fe6676e39ce7d4a0b03e439` 기준으로 반영됐다.
- `coupler-api` 운영 배포 후 내부/외부 응답, DB-backed 공개 API, PM2 상태, 로그를 확인했다.
- 운영 검증 완료 후 `coupler-api` `v2.2.2` annotated tag를 생성하고 원격에 push했다.
- `coupler-api`는 프로필 사진 승인 알림이 이미지별로 중복 발송되지 않도록, 프로필 세트 상태가 승인으로 전이될 때만 `profile_photo_ok` 승인 도메인을 기록한다.
- `coupler-api`는 설정 승인 푸시 문구에 `profile_photo_ok`를 추가해 프로필 사진 승인과 기본정보 승인을 구분한다.
- `coupler-mobile-app`은 native metadata 변경 없이 JS/locale/test/package script 변경만 포함하므로 Mobile NextPush-only로 Android `Production v20`, iOS `Production v17`을 배포했다.
- `coupler-admin-web`, Mobile Store, 운영 `RDS`는 이번 릴리스 범위에 포함하지 않는다.

## 메인 흐름

1. 배포 범위를 `coupler-api`, `Mobile NextPush`, `docs`, `Tag/Release Record`로 고정했다.
2. `coupler-api` 로컬 품질 게이트를 확인했다.
3. 운영 `coupler-api`를 `origin/main`의 `0d3532141f82c8582fe6676e39ce7d4a0b03e439`까지 fast-forward 반영했다.
4. 운영 `coupler-api`에서 `sudo env PATH="$PATH" pnpm install --frozen-lockfile`, `sudo pm2 restart coupler-api --update-env`, `sudo pm2 save`를 수행했다.
5. 운영 API 내부/외부 응답, DB-backed 공개 API, PM2 상태, 로그를 확인했다.
6. 운영 검증 완료 후 `coupler-api` `v2.2.2` annotated tag를 생성하고 원격에 push했다.
7. `coupler-mobile-app` 로컬 품질 게이트와 native 변경 없음 기준을 확인했다.
8. `coupler-mobile-app` Android/iOS NextPush Production 배포를 수행했다.
9. NextPush Android/iOS Production history에서 새 label, target binary version, package hash, rollout/mandatory/disabled 상태를 확인했다.
10. 이 문서를 운영 결과로 갱신하고 docs `v2.2.2` 태그/GitHub Release를 생성했다.

- `coupler-api`: 운영 배포 후 문제가 확인되면 직전 운영 태그 `v2.2.1` 또는 운영 배포 전 PM2 상태 기준으로 rollback한다.
