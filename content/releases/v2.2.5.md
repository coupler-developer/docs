# 2.2.5 릴리스 실행 기록

## 문서 역할

- 역할: `시나리오`
- 문서 종류: `flow`
- 충돌 시 우선 문서: `policy/release-process.md`, 태그 기준은 `policy/release-tag-policy.md`
- 기준 성격: `as-is`

## 목적

- `v2.2.5` docs 릴리스 기록에 API/Admin/Mobile 공통 응답과 ErrorData contract cutover 진행 기준점을 고정한다.
- 운영 API가 legacy envelope에서 공통 `{ ok, data | error }` envelope로 전환되는 시점과 검증 근거를 한 곳에서 추적한다.
- 코드 PR 준비 상태와 운영 cutover 완료 상태를 분리해 Draft 해제, merge, 배포, rollback 판단 기준을 명확히 한다.

## 범위

- 대상: `docs`, `coupler-api`, `coupler-admin-web`, `coupler-mobile-app`
- 포함 범위: 공통 응답 envelope, ErrorData contract, generated contract copy exact match, Mobile/Admin 응답 boundary, 강제 업데이트 차단 검증, Admin/API 운영 smoke 기준
- 제외 범위: 운영 `RDS` schema 변경, Store console 조작, NextPush console 조작, Codex의 운영 EC2 배포 수행

## 릴리스 결과

- 공통 응답/ErrorData contract cutover 코드는 PR 기준으로 준비됐지만 운영 반영은 완료되지 않았다.
- API PR #94는 `response_success`/`response_error` 공통 envelope 경계와 descriptor-first ErrorData 경계를 고정한다.
- Mobile PR #145와 Admin PR #47은 generated contract copy와 `ok` 기반 응답 boundary를 사용한다.
- 운영 `GET /app/auth/getSettingList?os=google&version_code=1`은 `app_info.force_update: 2`를 반환해 기존 낮은 build 차단 설정은 확인됐다.
- 같은 운영 응답은 아직 top-level `result_code`, `result_msg`, `result_data`를 반환하므로 API 공통 envelope cutover 완료 증거로 사용하지 않는다.

## 메인 흐름

1. API/Admin/Mobile/docs 공통 계약 PR의 CI와 mergeability를 확인한다.
2. PR 리뷰 승인 후 Draft를 해제하고 정책에 따라 `main`으로 병합한다.
3. API/Admin 운영 배포와 Mobile Store 또는 NextPush 적용 범위를 확정한다.
4. 운영 `getSettingList`에서 낮은 build가 `force_update: 2`로 차단되는지 확인한다.
5. 운영 API 응답이 `{ ok: true, data }` 또는 `{ ok: false, error }` envelope로 전환됐는지 확인한다.
6. Admin 운영자 액션 smoke와 Mobile 앱 기본 진입을 검증한다.
7. rollback 기준과 서비스 태그를 확정한 뒤 docs 릴리스 기록을 `released` 또는 실제 상태로 갱신한다.

### API contract cutover Gate

API/Admin/Mobile 공통 응답 또는 ErrorData contract cutover가 포함되므로 아래 항목을 모두 추적한다.
값을 확인하지 못한 항목은 `pending`으로 남기고 cutover 완료로 판정하지 않는다.

## 롤백 기준

- API/Admin/Mobile PR을 `main`에 병합하기 전에는 각 PR branch head를 rollback 기준으로 쓰지 않는다.
- 운영 배포 전 문제가 확인되면 Draft PR 상태를 유지하고 원인 수정 후 CI와 정책 검증을 재실행한다.
- 운영 배포 후 문제가 확인되면 직전 운영 API/Admin/Mobile SHA 또는 tag 기준으로 되돌린다.
- 공통 envelope 배포 후 legacy 앱에서 기본 진입 실패가 확인되면 cutover 완료로 닫지 않고 호환 배포 기준으로 복구한다.
- DB schema contract/drop은 이번 릴리스 범위가 아니며, 별도 DBM Gate 없이 실행하지 않는다.

## 후속 작업

- 각 PR의 리뷰 승인과 Draft 해제를 완료한다.
- API/Admin/Mobile 병합 후 운영 배포를 수행한다.
- 운영 API의 top-level envelope이 `{ ok, data | error }`로 전환됐는지 확인한다.
- Mobile Store 또는 NextPush 적용 근거와 Admin 운영자 액션 smoke를 추가한다.
- rollback SHA/tag와 서비스 태그 생성 결과를 기록한다.
