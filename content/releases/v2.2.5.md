## 릴리스 결과

- 공통 응답/ErrorData contract cutover 코드를 운영 API/Admin과 Mobile NextPush에 반영했다.
- API main은 `response_success`/`response_error` 공통 envelope 경계와 descriptor-first ErrorData 경계를 고정했고, 운영 `getSettingList` 응답의 top-level key가 `ok`, `data`로 확인됐다.
- Mobile/Admin main은 `@coupler-developer/coupler-api-contracts@0.1.1` dependency와 `ok` 기반 응답 boundary를 사용한다.
- 개발계 DB는 `75_expand_admin_manager_nickname.sql`/`76_postcheck_admin_manager_nickname.sql` 적용과 ledger 기록이 확인됐다.
- 운영 DB는 `75/76` 적용과 prod ledger 기록을 완료했고, `t_admin.nickname`이 `varchar(25) NOT NULL`로 확장됐다.
- 운영 `GET /app/auth/getSettingList?os=google&version_code=1`은 `{ ok: true, data }` envelope와 `app_info.force_update: 2`를 반환해 기존 낮은 build 차단 설정도 확인됐다.
- Admin 운영 배포 중 최초 백업 명령 quote 오류로 root 경로에 정적 파일이 복사되고 `/` owner가 변경된 이슈가 있었으나, 즉시 `/` owner를 `root:root`로 복구하고 root-level 정적 파일을 제거했으며 `/var/www/coupler-admin-web`와 외부 Admin 200 응답을 재확인했다.
