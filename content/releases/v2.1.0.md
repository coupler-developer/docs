- `coupler-mobile-app` 태그: `N/A` (`v2.1.0` 모바일 릴리즈 태그 미생성, 후속 `v2.2.0`으로 대체)

## 릴리스 결과

- `coupler-api` 운영 EC2를 `09dca9955cb9bbe49cb9a49584afc6ef8be627ec` 기준으로 배포하고 외부 응답을 확인했다.
- `coupler-admin-web` 운영 정적 산출물을 `ea5e017fc92cf446e6a7df49e7c7fc975fe6817d` 기준으로 배포하고 외부 응답을 확인했다.
- Android/iOS 스토어 빌드는 `2.1.0 (90)`으로 제출했다.
- Mobile Store `2.1.0 (90)`은 운영 출시/검증 완료 기준점으로 닫지 않았고, `coupler-mobile-app` `v2.1.0` 태그는 생성하지 않았다.
- 후속 `v2.2.0`에서 Store 승인/출시 기준점과 `coupler-mobile-app` `v2.2.0` 태그를 확정했으므로 이 릴리스의 모바일 대기 범위는 `superseded`로 종료한다.
- 이번 릴리스에서 운영 `RDS` 변경과 Mobile NextPush 배포는 포함하지 않는다.

## 메인 흐름

1. 개발계 API/Admin 배포를 완료한 뒤 운영계 API/Admin 배포 범위를 확정했다.
2. 운영 `coupler-api`에서 최신 `main` 반영, 의존성 설치, PM2 restart/save를 수행했다.
3. 운영 `coupler-admin-web`에서 최신 `main` 반영, production build, 정적 산출물 교체, nginx 검증/재시작을 수행했다.
4. Mobile Android AAB와 iOS archive를 `2.1.0 (90)`으로 스토어 심사에 제출했다.
5. API/Admin 운영 검증이 완료된 커밋에 서비스 태그를 생성하고, docs 릴리스 기록으로 완료/대기 범위를 분리했다.
6. 후속 `v2.2.0` 릴리스에서 모바일 Store 승인/출시 기준점이 확정돼, 이 문서의 모바일 대기 범위를 `superseded`로 닫았다.
