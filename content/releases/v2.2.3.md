## 릴리스 결과

- `coupler-admin-web`는 운영 EC2 정적 산출물을 `e7beb8f7ee5d3c79cca82530e5823b7e51d310c1` 기준 build로 교체했다.
- `coupler-admin-web`는 `member_review_status.key/display_state=input_required` 계약을 인식하고, 소개글 미입력 상태를 "소개글 입력 필요"로 표시한다.
- `coupler-api`는 운영 EC2에 `ae7b81558ee01c86a6d19a9a98bd6c288d6a0a0d` 기준으로 반영됐다.
- `coupler-api`는 소개글 미제출 Admin DTO를 `input_required` 계약으로 내려, 기존 `edit_need` 액션 필요 상태와 분리한다.
- `coupler-mobile-app`은 native metadata 변경 없이 JS-only 변경을 NextPush Android `Production v21`, iOS `Production v18`로 배포했다.
- 운영 `RDS`와 Mobile Store는 이번 릴리스 범위에 포함하지 않는다.

## 메인 흐름

1. 배포 범위를 `coupler-admin-web`, `coupler-api`, `Mobile NextPush`, `docs`, `Tag/Release Record`로 고정했다.
2. 계약 호환 순서를 고려해 `coupler-admin-web`를 먼저 운영 배포했다.
3. `coupler-admin-web` 로컬 품질 게이트와 production build를 완료했다.
4. 운영 EC2의 기존 Admin 정적 산출물을 `/var/www/coupler-admin-web-backup-20260705213140`로 백업했다.
5. `coupler-admin-web/build/` 산출물을 `/var/www/coupler-admin-web/`로 업로드하고 내부/외부 응답을 확인했다.
6. 운영 `coupler-api`를 `origin/main`의 `ae7b81558ee01c86a6d19a9a98bd6c288d6a0a0d`까지 fast-forward 반영했다.
7. 운영 `coupler-api`에서 `pnpm install --frozen-lockfile`, `pm2 restart coupler-api --update-env`, `pm2 save`를 수행했다.
8. 운영 API 내부/외부 응답, DB-backed 공개 API, PM2 상태, 로그를 확인했다.
9. `coupler-mobile-app` Android/iOS NextPush Production 배포를 수행했다.
10. NextPush Android/iOS Production history에서 새 label, target binary version, package hash, rollout/mandatory/disabled 상태를 확인했다.
11. 운영 검증 완료 후 `coupler-admin-web` `v2.1.3`, `coupler-api` `v2.2.3` annotated tag를 생성하고 원격에 push했다.
12. 이 문서를 운영 결과로 갱신하고 docs `v2.2.3` 태그/GitHub Release의 기준 문서로 확정했다.
