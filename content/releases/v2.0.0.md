## 실행 기록

1. `docs`에서 `v2.0.0` 태그를 먼저 생성해 GitHub Release Note 초안을 열었다.
2. `ritzy운영-coupler운영_마이그레이션_가이드/25_EXECUTION_PROCEDURE.md` 기준으로 `운영 dump baseline -> local full replay -> 개발계 확인 -> 운영 반영` 순서를 적용했다.
3. 운영 `RDS`에서 `44`, `45`를 순서대로 실행했고 두 SQL 모두 `DBM-GATE-400 OK`로 종료됐다.
4. `coupler-api`, `coupler-admin-web`, `coupler-mobile-app`를 실제 배포 기준 커밋으로 확정하고 `v2.0.0` 태그를 생성했다.
5. `v2.0.0` 릴리스 문서를 계획형 문서가 아니라 실행 기록으로 닫아, 태그/SHA/Gate/log 근거를 한곳에 남겼다.
