# 2.2.4 릴리스 실행 기록

## 문서 역할

- 역할: `시나리오`
- 문서 종류: `flow`
- 충돌 시 우선 문서: `policy/release-process.md`, 태그 기준은 `policy/release-tag-policy.md`
- 기준 성격: `as-is`

## 목적

- `v2.2.4` docs 릴리스 기록에 Mobile Store `2.2.1 (100)` 배포 진행 기준점을 고정한다.
- docs 릴리스 번호는 Mobile Store 버전 번호를 강제하지 않고, 배포 기록의 순번으로 사용한다.
- Store archive, upload, review, release, 제출 마커 증빙, 최종 모바일 릴리스 태그를 배포 진행 상태와 분리해 추적한다.

## 범위

- 대상: `docs`, `coupler-mobile-app`
- 포함 범위: Mobile Store `2.2.1 (100)` native metadata merge 기준, PR/CI 검증 결과, Store 배포 진행 기록, docs Release 기록
- 제외 범위: 운영 `RDS` 변경, `coupler-api` 운영 배포, `coupler-admin-web` 운영 배포, Mobile NextPush 배포, Codex의 Store archive/upload/승인/출시 수행

## 릴리스 결과

- `coupler-mobile-app` PR #143이 `main`에 merge되어 Store binary용 native metadata가 `2.2.1 (100)` 기준으로 준비됐다.
- Android는 `versionName 2.2.1`, `versionCode 100`으로 갱신됐다.
- iOS는 `MARKETING_VERSION 2.2.1`, `CURRENT_PROJECT_VERSION 100`으로 갱신됐다.
- Codex는 Store archive, Android AAB `2.2.1 (100)` 생성, iOS archive 생성, Store upload/submission/review/release를 수행하지 않았다.
- 기존 로컬 Android AAB `android/app/build/outputs/bundle/release/2.2.1(99).aab`는 목표 build `100`과 다르므로 이번 Store 제출 산출물로 사용하지 않는다.
- Mobile NextPush는 이번 릴리스 범위에 포함하지 않는다.

## 메인 흐름

1. Mobile Store `2.2.1 (100)`을 `v2.2.3` Admin/API 릴리스와 분리된 Store 릴리스 범위로 확정했다.
2. `coupler-mobile-app` release PR #143에서 Android/iOS native metadata를 `2.2.1 (100)`으로 갱신했다.
3. PR #143의 local quality gate와 GitHub Actions를 확인했다.
4. PR #143을 `main`에 merge해 Store build 기준 커밋을 확정했다.
5. Codex는 Store archive/upload 단계에 개입하지 않고, Store 배포 진행 상태를 docs `v2.2.4`에 기록한다.
6. Store archive와 제출은 릴리즈 작업자가 별도로 수행한다.
7. Store 제출 직후 제출 마커 태그 생성 여부와 산출물 hash를 기록한다.
8. Store 승인/운영 출시/기본 검증 완료 후 `coupler-mobile-app` 릴리스 태그와 docs 최종 상태를 갱신한다.

## 롤백 기준

- Store 제출 전 문제가 확인되면 `coupler-mobile-app` `main`에서 PR #143 merge 커밋을 revert하거나 후속 version/build 수정 PR을 만든다.
- Store 제출 후 문제가 확인되면 Store console의 제출 취소, 새 build 업로드, 단계적 출시 중단, 또는 후속 hotfix build 제출 중 실제 상태에 맞는 조치를 선택한다.
- Store 승인/운영 출시 전에는 `coupler-mobile-app` `v2.2.1` 릴리스 태그를 생성하지 않는다.
- 제출 마커 태그를 생성한 뒤 Store 제출을 중단하면 제출 마커 태그 삭제 또는 보류 사유를 docs 최종 기록에 남긴다.
- 운영 `RDS`, `coupler-api`, `coupler-admin-web`, Mobile NextPush는 이번 릴리스 롤백 범위가 아니다.

## 후속 작업

- Store archive 산출물 경로와 hash를 기록한다.
- Android AAB `2.2.1 (100)` 산출 여부를 기록한다.
- iOS archive/App Store Connect upload 여부를 기록한다.
- Store 제출/승인/운영 출시 시각과 검증 결과를 기록한다.
- 제출 마커 태그 `submitted/mobile-2.2.1-100` 생성 여부와 tag commit을 기록한다.
- Store release 후 모바일 `v2.2.1` 릴리스 태그 생성 여부를 확정한다.
- docs `v2.2.4` 기록을 `released` 또는 다른 최종 상태로 갱신한다.
