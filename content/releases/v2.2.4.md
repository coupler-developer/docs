## 목적

- `v2.2.4` docs 릴리스 기록에 Mobile Store `2.2.1 (100)` 배포 진행 기준점을 고정한다.

- `coupler-mobile-app` PR #143이 `main`에 merge되어 Store binary용 native metadata가 `2.2.1 (100)` 기준으로 준비됐다.

- Codex는 Store upload/submission/review/release를 수행하지 않았고, 사용자가 Store upload/submission/review 제출 완료를 보고했다.
- 사용자가 Store review 승인 완료를 보고했다. 승인 확인 시각은 사용자 보고 기준 `2026-07-08 15:07:55 KST`로 기록한다.

- `coupler-mobile-app` 릴리스 태그 `v2.2.1`을 Store binary 기준 커밋 `3efcfaec3de2f37210be69454d95bc7f5eb88ad6`에 생성하고 원격에 push했다.
- 제출 마커 태그 `submitted/mobile-2.2.1-100` 증빙을 본 기록으로 이관하고 로컬/원격에서 삭제했다.
- Android AAB `2.2.1 (100)` 산출물은 `android/app/build/outputs/bundle/release/2.2.1(100).aab`로 확인됐다.

1. Mobile Store `2.2.1 (100)`을 `v2.2.3` Admin/API 릴리스와 분리된 Store 릴리스 범위로 확정했다.
2. `coupler-mobile-app` release PR #143에서 Android/iOS native metadata를 `2.2.1 (100)`으로 갱신했다.
3. PR #143의 local quality gate와 GitHub Actions를 확인했다.
4. PR #143을 `main`에 merge해 Store build 기준 커밋을 확정했다.
5. Android AAB `2.2.1 (100)`와 iOS archive `2.2.1 (100)` 산출물을 확인했다.
6. 사용자가 Store upload/submission/review 제출 완료를 보고했다.
7. Store 제출 마커 태그 `submitted/mobile-2.2.1-100`를 생성하고 원격에 push했다.
8. 사용자가 Store review 승인 완료를 보고했다.
9. 사용자가 Store 운영 출시와 기본 검증 완료를 보고했다.
10. `coupler-mobile-app` 릴리스 태그 `v2.2.1`을 생성하고 원격에 push했다.
11. 제출 마커 태그 `submitted/mobile-2.2.1-100` 증빙을 릴리스 기록으로 이관하고 로컬/원격에서 삭제했다.
12. docs 릴리스 기록을 최종 `released` 상태로 갱신했다.

## 롤백 기준

- Store 제출 전 문제가 확인되면 `coupler-mobile-app` `main`에서 PR #143 merge 커밋을 revert하거나 후속 version/build 수정 PR을 만든다.
- Store 출시 후 문제가 확인되면 Store console의 단계적 출시 중단, 새 build 업로드, 또는 후속 hotfix build 제출 중 실제 상태에 맞는 조치를 선택한다.
- `coupler-mobile-app` `v2.2.1` 태그는 Store `2.2.1 (100)` 출시 기준점이다.
- 운영 `RDS`, `coupler-api`, `coupler-admin-web`, Mobile NextPush는 이번 릴리스 롤백 범위가 아니다.
