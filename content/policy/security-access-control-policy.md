# 보안/접근통제 정책

## 문서 역할

- 역할: `규범`
- 문서 종류: `policy`
- 충돌 시 우선 문서: 이 문서
- 기준 성격: `as-is`

## 목적

- 인증/인가/권한 변경 흐름을 단일 기준으로 고정해 권한 우회와 운영 실수를 방지한다.

## 적용 범위

- `coupler-api`, `coupler-admin-web`, `coupler-mobile-app`
- 관리자 기능, 운영 액션(승인/반려/제재/무료키 지급), 회원 민감 데이터 조회/수정

## 필수 규칙

### 1) 인증/인가 책임 분리

- 인증(누구인가)은 토큰/세션 검증 미들웨어에서만 수행한다.
- 인가(무엇을 할 수 있는가)는 API 서버에서만 판정한다.
- 클라이언트(Admin/Mobile)는 표시 제어만 수행하고 권한 판정을 우회하지 않는다.

### 2) 서버 단일 판정

- 비즈니스 상태 전이/권한 판정은 API 서버를 단일 기준으로 사용한다.
- Admin UI에서 버튼 숨김 여부와 무관하게 서버에서 동일하게 차단해야 한다.
- `req.admin.super`/담당자 여부 같은 권한 조건은 액션 단위로 명시한다.

#### Admin 그룹미팅 메뉴 노출

- 기존 2:2 `그룹미팅 관리` 상위 메뉴와 그 하위 5개 메뉴는 Super Admin에게만 노출하고, 일반 매니저
  사이드바에서는 숨긴다. Admin route의 상위 메뉴에 `manager: true`를 적용한다.
- 신규 N:N `클럽 Host 단체미팅 관리` 메뉴는 Super Admin과 일반 매니저 모두에게 노출한다. 일반 매니저의
  행사 조회·변경 범위는 서버가 자신이 소유한 행사로 제한하고, Super Admin만 전체 운영 범위를 가진다.
- 메뉴 노출 여부는 클라이언트 표시 규칙이며 API 인가를 대신하지 않는다. 직접 URL 또는 API 요청의
  허용·거부는 각 서버 operation의 인증·인가 계약으로 판정하고, Admin route flag를 권한 근거로 사용하지
  않는다.

### 3) 운영 액션 통제

- 승인/반려/제재/무료키 지급 등 운영 액션은 사유 입력을 필수로 강제한다.
- 중요 액션은 `actor`, `target`, `reason`, `timestamp`와 영속 감사 레코드 식별자(`audit_id` 등)를
  함께 기록한다. 요청 계층에 canonical correlation ID가 이미 있으면 구조화 로그에도 함께 기록하되,
  추적 목적만으로 감사 원장에 `request_id` 컬럼을 추가하지 않는다. 중복 실행 방지가 필요하면 별도
  idempotency key와 저장소 유일성 제약을 명시적으로 설계한다.
- 권한 상향(부여)과 권한 하향(회수)은 이력 로그를 남기고, 로그 없는 변경은 금지한다.

### 4) 실패 처리 (Fail-closed)

- 권한 정보 누락/불일치/미정의 상태는 기본 허용이 아니라 즉시 거부한다.
- 예외 분기(`if (...) return;`)로 조용히 통과시키는 코드를 금지한다.
- 권한 실패 응답은 [API 에러 계약 정책](api-error-contract-policy.md)의 `ErrorData`로 반환하고 서버 로그를 남긴다.

### 5) 예외/임시 권한 규칙

- 임시 권한 부여는 담당자, 만료 시점, 추적 이슈를 명시한 경우만 허용한다.
- 만료 조건 없는 임시 권한은 배포 전 차단한다.
- 임시 권한 종료 후 회수 검증 로그를 PR/작업 보고에 첨부한다.

## 운영 절차

1. 권한 변경 요청: 변경 대상/사유/영향 범위를 기록한다.
2. 사전 검토: API 인가 로직, UI 노출 로직, 로그 스키마 변경 여부를 점검한다.
3. 적용: 서버 인가 로직을 먼저 반영하고 클라이언트 표시 로직을 동기화한다.
4. 검증: 권한별 허용/거부 시나리오를 실행해 기대 결과를 확인한다.
5. 롤백: 비정상 권한 노출 시 직전 태그/커밋 기준으로 즉시 복구한다.

## 증빙/추적

- PR 본문에 아래를 필수로 남긴다.
    - 권한 매트릭스 변경 전/후
    - 허용/거부 테스트 결과(로그 또는 캡처 링크)
    - Super Admin·일반 매니저별 메뉴 노출 결과
    - 문서 동기화 반영 경로
- 운영 반영 시 감사 로그 조회 경로를 릴리즈 노트에 명시한다.
