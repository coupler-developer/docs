# Technical Debt 템플릿

## 문서 역할

- 역할: `부채`
- 문서 종류: `technical-debt`
- 충돌 시 우선 문서: 해당 항목이 참조하는 규범 문서
- 기준 성격: `as-is`

## 목적

- 미해결 기술 부채만 기록한다.

## 적용 범위

- 평가 대상 레포/서비스 범위를 명시한다.

## 평가 기준

- 코드 현상은 최신 clean `main`, 운영 현상은 최신 released 릴리스 기록을 기준으로 판정한다.
- 열린 브랜치나 PR의 구현은 완료 근거로 사용하지 않는다.
- 시점에 따라 바뀌는 API 계약 package의 concrete version은 적지 않는다. 현재 정렬은 package manifest·lockfile의 생성 가능한
  비교 결과로 판정하고, 특정 릴리스의 exact version과 기준 ref는 릴리스 기록에 둔다.
- 우선순위 `P1`은 운영·릴리즈·보안·계약 차단 위험, `P2`는 사용자 일관성·유지보수 위험,
  `P3`는 명시적으로 보류 가능한 정리 범위를 뜻한다.
- 작업량 `S`는 단일 레포의 작은 PR, `M`은 다수 파일 또는 교차 검증이 필요한 작업,
  `L`은 다중 레포·단계적 cutover·운영 검증이 필요한 작업을 뜻한다.

- 계약 package 정렬 부채는 concrete version 없이 미해결 조건과 완료 Gate만 기록한다.
- 완료 시 항목을 삭제하고 이력은 PR·릴리스 기록에 둔다.
