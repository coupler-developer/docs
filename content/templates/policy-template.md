# Policy 템플릿

## 목적

- 이 정책이 해결하려는 운영/품질 문제를 한 줄로 명시한다.

## 적용 범위

- 적용 레포/서비스/운영 환경을 명시한다.
- 제외 범위가 있으면 함께 적는다.

## 단일 SoT

- 이 정책이 참조하는 상위 기준 문서를 명시한다.
- 중복 규칙을 만들지 않고 책임 경계를 분리한다.

## 필수 규칙

- MUST 규칙을 명시한다.
- SHOULD 규칙을 명시한다.
- 예외 허용 조건과 만료 조건을 명시한다.

## 운영 절차

- 변경 요청 -> 검토 -> 적용 -> 검증 -> 롤백 순서를 명시한다.

## 증빙/추적

- PR/로그/대시보드/문서에 남겨야 할 증빙 항목을 명시한다.

## 체크리스트

- [ ] 적용 전 점검 항목
- [ ] 적용 후 검증 항목
- [ ] 실패 시 롤백 항목

## 관련 문서

- 연관 architecture/flow/policy 문서 링크를 나열한다.
